‚ö° Quick Start

The fastest way to get PersonalJARVIS running. Three commands, and you're ready.

Prerequisites

Make sure you have Node.js 18+ installed. You can check with:

$ node --version
v20.11.0

Install & Setup

# 1. Clone the repository
$ git clone https://github.com/hamzafire224-arch/JARVIS-OS.git
$ cd JARVIS-OS

# 2. Install dependencies
$ npm install

# 3. Run the setup wizard
$ npm run setup

# Or use quick setup with defaults (Gemini + Security)
$ npm run setup -- --quick

Start JARVIS

# Start with preflight checks
$ npm run quick-start

# Or start directly
$ npm start

That's it! JARVIS will start and you can begin interacting with your AI assistant.

üõ°Ô∏è Security

PersonalJARVIS enforces capability-based security on every tool operation.

Security Presets

Strict

Approve everything. Maximum safety for untrusted environments.

Balanced (Default)

Auto-approve safe ops. Prompt for moderate+. Best for most users.

Developer

Auto-approve safe + moderate. Only prompt for dangerous ops.

Trust

Auto-approve everything. Only for trusted, isolated environments.

How It Works

Every tool call passes through the CapabilityManager which checks risk level, blocked paths/commands, and your active policy. All operations are logged to an audit trail.

// The flow for every tool execution:
checkPermission('run_command', args)
  ‚Üí assessRisk() ‚Üí 'dangerous'
  ‚Üí checkBlockedPatterns() ‚Üí pass
  ‚Üí applyPolicy('balanced') ‚Üí prompt user
  ‚Üí logAudit() ‚Üí saved

üß† Memory System

PersonalJARVIS uses a 4-layer hierarchical memory system to maintain context across conversations.

Memory Layers

Layer Purpose Retention
Working Current conversation messages Session only
Episodic Session summaries & highlights 30 days
Semantic User facts & preferences Permanent
Vector Similarity search & reranking Permanent

On the Balanced plan, you get Working + Semantic memory. The Productivity plan unlocks all 4 layers including episodic session summaries and vector-based retrieval.

üìä Analytics

Track your productivity gains and cost savings ‚Äî all locally, all private.

The analytics system tracks sessions, messages, tool usage, and model routing decisions. Data is automatically anonymized and stored on your machine.

What's Tracked

// Event types tracked by UsageAnalytics:
'session_start' | 'session_end'
'message_sent' | 'message_received'
'tool_used' | 'model_routed'
'memory_saved' | 'error'

Analytics data flushes every 5 minutes and is retained for 90 days by default. All personal content is anonymized before storage.

‚öôÔ∏è Configuration

PersonalJARVIS stores its configuration in two files at the project root.

.env

# AI Provider
JARVIS_PROVIDER=gemini
GEMINI_API_KEY=your-key-here

# Security
JARVIS_SECURITY_ENABLED=true
JARVIS_SECURITY_PRESET=balanced

# Tiered Inference
JARVIS_TIERED_INFERENCE=true
JARVIS_OLLAMA_MODEL=llama3.2

jarvis.config.json

{
  "provider": "gemini",
  "dataDir": "./data",
  "security": { "enabled": true, "preset": "balanced" },
  "inference": { "tiered": true },
  "persona": "professional"
}
