Quick Start

Get PersonalJARVIS up and running in under 5 minutes.

Prerequisites

You need Node.js 18+ installed. That's it.

1. Install

# Install globally via npm
$ npm install -g personaljarvis

2. Run Setup

# Interactive setup wizard
$ jarvis setup

# Or quick setup with defaults
$ jarvis setup --quick

The wizard will guide you through choosing an AI provider, entering your API key, and configuring security settings.

3. Start JARVIS

# Start the interactive REPL
$ jarvis

🤖 JARVIS v1.0.0 — Productivity Plan
Type your request or /help for commands

You: What files are in this project?

AI Providers

JARVIS supports multiple AI backends. You can switch between them at any time.

Google Gemini (Recommended)

Best for beginners. Has a free tier.

# In your .env file
JARVIS_PROVIDER=gemini
GEMINI_API_KEY=your-key-here

# Get a free key at:
# https://aistudio.google.com/apikey

OpenAI (GPT-4)

JARVIS_PROVIDER=openai
OPENAI_API_KEY=your-key-here

Anthropic (Claude)

JARVIS_PROVIDER=anthropic
ANTHROPIC_API_KEY=your-key-here

Ollama (Local, Free)

Run AI entirely on your machine. No API key needed.

# Install Ollama from https://ollama.ai
$ ollama pull llama3:8b

# In your .env
JARVIS_PROVIDER=ollama
OLLAMA_MODEL=llama3:8b

Security

JARVIS has a built-in security framework that protects your system.

What's Protected

Blocked paths: ~/.ssh, ~/.aws, ~/.gnupg, system directories, and more.

Blocked commands: rm -rf /, sudo rm, format, and other destructive operations.

Audit logging: Every tool execution is logged with timestamp, command, and result.

Configuration

# In your .env
JARVIS_SECURITY_ENABLED=true

# Security modes: trust, balanced, strict
# Default is balanced

Memory System

JARVIS has a 4-layer hierarchical memory that persists across sessions.

Layers

Working Memory: Current session context. Cleared on restart.

Episodic Memory: Past conversations and interactions. Searchable by topic.

Semantic Memory: Facts, preferences, and learned knowledge about your patterns.

Vector Memory: Similarity search for finding related past experiences.

Preference Learning

JARVIS automatically learns your coding style, indentation preferences, naming conventions, and test patterns. These are stored in ~/.jarvis/preferences.json and applied to all future interactions.

Skills

Extend JARVIS with installable skills from the marketplace or custom-built ones.

Built-in Commands

/install <skill-name> # Install a skill
/skills # List installed skills
/uninstall <skill> # Remove a skill
/connect <mcp-tool> # Connect via MCP

Creating Custom Skills

Skills are TypeScript modules that export a Skill interface. See the skills directory for examples.

CLI Reference

Commands

jarvis # Start interactive REPL

jarvis setup --quick # Quick setup with defaults
jarvis --version # Show version
jarvis --help # Show help

In-Session Commands

/help # Show available commands
/status # Show license & plan status
/memory # Show memory stats
/skills # List installed skills
/clear # Clear working memory
/exit # Quit JARVIS

License Key

Your license key links JARVIS to your PersonalJARVIS account.

Getting a Key

1. Create an account at app.personaljarvis.dev/signup

2. Go to the License page in your dashboard

3. Click Generate License Key

4. Copy the key and enter it in jarvis setup

Key Management

Your key is stored in ~/.jarvis/config.json. It validates against our API every 24 hours and caches the result. Works offline for up to 24 hours between checks.
